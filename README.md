# ГородОК

Система анализа обращений граждан из Excel: очистка данных, определение проблем и серьёзности, отчёты Top-3 / Top-10 по районам, PDF-справка для руководства и веб-дашборд Streamlit.

## Возможности

- **Пайплайн обработки** — от сырого Excel до аналитики и отчётов (несколько режимов: правила, кластеризация + LLM, полный LLM).
- **Дашборд** — KPI, таблица обращений, карта Омской области, графики, подсветка муниципалитета.
- **Сравнение отчётов** — два периода рядом, дельты по районам, экспорт в PDF.
- **История отчётов** — переключение между папками `Отчет от …` в боковой панели.
- **Скачивание** — полный Excel, топ районов, PDF-справка, ZIP-архив.

## Требования

| Компонент | Минимум |
|-----------|---------|
| ОС | Windows 10/11 (скрипты `.ps1` / `.bat`) |
| Python | 3.11+ |
| GPU | Нужна для режимов с LLM (рекомендуется NVIDIA, CUDA) |
| Диск | ~8 ГБ под модель Qwen2.5-7B (кэш Hugging Face) |
| RAM | 16 ГБ+ для LLM в 4-bit |

Режимы **«Только очистка»** и **«Быстрый (правила)»** работают без GPU.

## Установка

Из корня репозитория:

```powershell
.\setup.ps1
```

Скрипт создаёт `.venv`, ставит PyTorch (CUDA 13.0) и зависимости из `requirements.txt`.

Затем загрузите языковую модель (один раз, нужна для LLM-режимов):

```powershell
.\.venv\Scripts\python.exe tools\download_model.py
```

Модель: **Qwen/Qwen2.5-7B-Instruct** (4-bit, скачивается в кэш Hugging Face).

Тестовый Excel положите в `storage\samples\` (файлы `.xlsx` в Git не попадают).

Альтернатива без PowerShell: `setup.bat`.

## Запуск дашборда

```powershell
.\run.ps1
```

Откроется Streamlit на `http://localhost:8501`. Альтернатива: `run.bat`.

Скрипт ищет виртуальное окружение в порядке: `.venv` → `qwen_env` (в проекте или у родительской папки).

**Работа в интерфейсе:**

1. В боковой панели загрузите `.xlsx`.
2. Выберите режим обработки (по умолчанию — **Кластерный + turbo**).
3. Нажмите **Запустить обработку**.
4. Переходите по разделам: **Главная**, **Графики**, **Отчёты**, **Сравнение**.

Подробнее о разделах: [docs/ДАШБОРД.md](docs/ДАШБОРД.md).

## Запуск из командной строки

Рекомендуемый режим для полного файла:

```powershell
.\.venv\Scripts\python.exe analyze\pipeline.py `
  --input "storage\samples\файл.xlsx" `
  --work-dir . `
  --mode cluster --turbo
```

Все режимы и параметры: [docs/РЕЖИМЫ_ОБРАБОТКИ.md](docs/РЕЖИМЫ_ОБРАБОТКИ.md).

## Результаты обработки

Папка: `storage\output\Отчет от дд.мм.гг - чч.мм\`

| Файл | Содержимое |
|------|------------|
| `… — очищенные данные.xlsx` | После фильтрации и нормализации |
| `… — полный анализ.xlsx` | Все строки с полями проблема / серьёзность |
| `… — топ районов.xlsx` | Листы: Топ-10, Топ-3, Все районы, Обзор |
| `… — справка.pdf` | PDF для руководства |
| `… — сводка.txt` | Краткий текстовый итог |
| `cleaning_stats.json` | Статистика очистки (для KPI в дашборде) |

Поддерживаются и старые имена (`result.xlsx`, `report.xlsx`, `report.pdf`).

Дашборд копирует активный отчёт в `dashboard\data\reports\` для отображения.

## Разделы дашборда

| Раздел | Назначение |
|--------|------------|
| **Главная** | Скачивание файлов, KPI, топ-3 района, таблица обращений |
| **Графики** | Карта, ТОП-10, динамика, подсветка муниципалитета |
| **Отчёты** | KPI, сводка для руководства, таблица |

```

```

## Режимы обработки (кратко)

| Режим | GPU | Назначение |
|-------|-----|------------|
| Только очистка | нет | Фильтрация и нормализация |
| Быстрый (правила) | нет | Ключевые слова, Top-3/Top-10 |
| Гибридный | да | Правила + LLM-описание топ-районов |
| Кластерный + turbo | да | **Рекомендуется** для больших файлов |
| Кластерный полный | да | Больше вызовов LLM |
| Полный LLM | да | Qwen на каждую строку |

## Устранение неполадок

**«Venv not found»** — выполните `.\setup.ps1`.

**«Python не найден» при обработке из дашборда** — проверьте наличие `.venv\Scripts\python.exe`.

**Модель не грузится / offline** — сначала запустите `tools\download_model.py` при доступе к интернету.

**Карта пустая** — завершите анализ и выберите отчёт в истории в сайдбаре.

**Сравнение недоступно** — нужно минимум два отчёта в `storage\output` с файлом «… — топ районов.xlsx».
