# Режимы обработки Excel

Режимы доступны в **боковой панели дашборда** и из CLI через `analyze/pipeline.py`.

## Сводная таблица

| Режим (UI) | CLI | GPU | Время (≈400k строк) | Результат |
|------------|-----|-----|---------------------|-----------|
| Только очистка | `excel.py` | нет | 1–3 мин | очищенные данные |
| Быстрый (правила) | `--mode fast` | нет | 2–5 мин | полный отчёт |
| Гибридный | `--mode hybrid` | да | 5–15 мин | правила + LLM на ТОП |
| Кластерный + turbo | `--mode cluster --turbo` | да | 10–25 мин | **рекомендуется** |
| Кластерный полный | `--mode cluster` | да | 45–90 мин | больше вызовов LLM |
| Полный LLM | `--mode llm` | да | часы–дни | каждая строка |

Время ориентировочное и зависит от GPU, диска и доли «решаемых» обращений после очистки.

## Этапы пайплайна

```
Исходный Excel
    → excel.py           (очистка, фильтр «Решаемый», дедупликация)
    → анализ              (fast / cluster / llm / hybrid)
    → aggregate.py        (Top-3, Top-10, PDF, TXT-сводка)
    → summarize_top.py    (только hybrid — LLM-описание топ-районов)
```

### Скрипты анализа

| `--mode` | Скрипт | Описание |
|----------|--------|----------|
| `fast` | `fast_analyze.py` | Правила по ключевым словам |
| `cluster` | `cluster_analyze.py` | Эмбеддинги + LLM на представителях кластеров |
| `llm` | `test_qwen.py` | Qwen на каждую строку |
| `hybrid` | `fast_analyze.py` + `summarize_top.py` | Правила, затем LLM для топ-районов |

Флаг `--turbo` ускоряет кластерный режим (~800 вызовов LLM, summary из темы кластера).

## Примеры CLI

### Рекомендуемый (полный файл)

```powershell
.\.venv\Scripts\python.exe analyze\pipeline.py `
  --input "storage\samples\файл.xlsx" `
  --work-dir . `
  --mode cluster --turbo
```

### Только очистка

```powershell
.\.venv\Scripts\python.exe analyze\excel.py `
  --input "storage\samples\файл.xlsx" `
  --output "storage\output\cleaned.xlsx"
```

### Быстрый (без нейросети)

```powershell
.\.venv\Scripts\python.exe analyze\pipeline.py `
  --input "storage\samples\файл.xlsx" `
  --work-dir . `
  --mode fast
```

### Гибридный

```powershell
.\.venv\Scripts\python.exe analyze\pipeline.py `
  --input "storage\samples\файл.xlsx" `
  --work-dir . `
  --mode hybrid
```

### Кластерный полный (без turbo)

```powershell
.\.venv\Scripts\python.exe analyze\pipeline.py `
  --input "storage\samples\файл.xlsx" `

.\.venv\Scripts\python.exe analyze\pipeline.py `

  --mode llm --batch-size 8
```

## Дополнительные параметры

| Параметр | Описание |
|----------|----------|
| `--limit N` | Обработать только N строк (после очистки) |
| `--random` | Случайная выборка при `--limit` |
| `--batch-size N` | Размер батча LLM (по умолчанию 8) |
| `--clusters N` | Верхний предел кластеров ≈ вызовов LLM (по умолчанию 800) |
| `--skip-clean` | Пропустить очистку |
| `--cleaned путь` | Готовый файл очищенных данных |
| `--skip-llm` | Пропустить анализ, только отчёт |
| `--analyzed путь` | Готовый полный анализ |
| `--output-dir путь` | Папка для результатов |

### Пример: выборка 1000 строк

```powershell
.\.venv\Scripts\python.exe analyze\pipeline.py `
  --input "storage\samples\файл.xlsx" `
  --work-dir . `
  --mode cluster --turbo `
  --limit 1000 --random
```

### Пример: продолжить с очищенного файла

```powershell
.\.venv\Scripts\python.exe analyze\pipeline.py `
  --work-dir . `
  --skip-clean `
  --cleaned "storage\output\Отчет от 01.01.26 - 12.00\Отчет от 01.01.26 - 12.00 — очищенные данные.xlsx" `
  --mode cluster --turbo
```

## Файлы результата

Папка: `storage/output/Отчет от дд.мм.гг - чч.мм/`

| Файл | Когда появляется |
|------|------------------|
| `… — очищенные данные.xlsx` | После очистки (или `--skip-clean` с копией) |
| `cleaning_stats.json` | После очистки |
| `… — полный анализ.xlsx` | После этапа анализа |
| `… — топ районов.xlsx` | Листы: Топ-10, Топ-3, Все районы, Обзор |
| `… — справка.pdf` | PDF для руководства |
| `… — сводка.txt` | Краткий текстовый итог |

Старые имена (`cleaned.xlsx`, `result.xlsx`, `report.xlsx`, `report.pdf`, `report.txt`) тоже поддерживаются.

Дашборд читает активный отчёт из `storage/output/…` и кэширует копии в `dashboard/data/reports/`.

## Модель

Для LLM-режимов нужна предварительная загрузка:

```powershell
.\.venv\Scripts\python.exe tools\download_model.py
```

Модель: **Qwen/Qwen2.5-7B-Instruct** (4-bit quantization).
