# Режимы обработки Excel

Все режимы доступны в **боковой панели дашборда** и из командной строки через `analyze/pipeline.py`.

## Сводная таблица

| Режим (UI) | CLI | GPU | Время (≈400k) | Результат |
|------------|-----|-----|---------------|-----------|
| Только очистка | `excel.py` | нет | 1–3 мин | `cleaned.xlsx` |
| Быстрый (правила) | `--mode fast` | нет | 2–5 мин | полный отчёт |
| Гибридный | `--mode hybrid` | да | 5–15 мин | правила + LLM на ТОП |
| Кластерный + turbo | `--mode cluster --turbo` | да | 10–25 мин | **рекомендуется** |
| Кластерный полный | `--mode cluster` | да | 45–90 мин | больше вызовов LLM |
| Полный LLM | `--mode llm` | да | часы–дни | каждая строка |

## Этапы пайплайна

```
Исходный Excel
    → excel.py          (очистка, фильтр «Решаемый», дедупликация)
    → анализ            (fast / cluster / llm / hybrid)
    → aggregate.py      (Top-3, Top-10, report.pdf)
    → summarize_top.py  (только hybrid — LLM-описание топ-районов)
```

## Примеры CLI

### Рекомендуемый (полный файл)

```powershell
.\.venv\Scripts\python.exe analyze\pipeline.py `
  --input "storage\samples\файл.xlsx" `
  --work-dir . `
  --mode cluster --turbo
```

### Только очистка

```powershell
.\.venv\Scripts\python.exe analyze\excel.py `
  --input "storage\samples\файл.xlsx" `
  --output "storage\output\cleaned.xlsx"
```

### Быстрый (без нейросети)

```powershell
.\.venv\Scripts\python.exe analyze\pipeline.py `
  --input "storage\samples\файл.xlsx" `
  --work-dir . `
  --mode fast
```

### Гибридный

```powershell
.\.venv\Scripts\python.exe analyze\pipeline.py `
  --input "storage\samples\файл.xlsx" `
  --work-dir . `
  --mode hybrid
```

### Кластерный полный (без turbo)

```powershell
.\.venv\Scripts\python.exe analyze\pipeline.py `
  --input "storage\samples\файл.xlsx" `
  --work-dir . `
  --mode cluster --clusters 800 --batch-size 8
```

### Полный LLM (каждая строка)

```powershell
.\.venv\Scripts\python.exe analyze\pipeline.py `
  --input "storage\samples\файл.xlsx" `
  --work-dir . `
  --mode llm --batch-size 8
```

## Дополнительные параметры

| Параметр | Описание |
|----------|----------|
| `--limit N` | Обработать только N строк (после очистки) |
| `--random` | Случайная выборка при `--limit` |
| `--batch-size N` | Размер батча LLM (по умолчанию 8) |
| `--clusters N` | Верхний предел кластеров ≈ вызовов LLM (по умолчанию 800) |
| `--skip-clean` | Пропустить очистку |
| `--cleaned путь` | Готовый `cleaned.xlsx` |
| `--skip-llm` | Пропустить анализ, только отчёт |
| `--analyzed путь` | Готовый `result.xlsx` |
| `--output-dir путь` | Папка для результатов |

### Пример: выборка 1000 строк

```powershell
.\.venv\Scripts\python.exe analyze\pipeline.py `
  --input "storage\samples\файл.xlsx" `
  --work-dir . `
  --mode cluster --turbo `
  --limit 1000 --random
```

### Пример: продолжить с очищенного файла

```powershell
.\.venv\Scripts\python.exe analyze\pipeline.py `
  --work-dir . `
  --skip-clean `
  --cleaned "storage\output\Отчет от …\cleaned.xlsx" `
  --mode cluster --turbo
```

## Файлы результата

Папка `storage/output/Отчет от дд.мм.гг - чч.мм/`:

| Файл | Когда появляется |
|------|------------------|
| `cleaned.xlsx` | Всегда (кроме `--skip-clean` без копии) |
| `result.xlsx` | После этапа анализа |
| `report.xlsx` | Top-3 / Top-10 |
| `report.pdf` | PDF-справка для руководства |

Дашборд читает копии из `dashboard/data/reports/`.
